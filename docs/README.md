# Sandi Docs

Sandi's docs are split by relationship to the project.

- [Sandi for partners](partners/README.md): how people interact with Sandi day
  to day, what she can do in shared spaces, and how to shape a live Sandi's
  personality, memory, and habits.
- [Sandi for developers](developers/README.md): how Sandi is built, configured,
  deployed, extended, and verified.

If you are deciding where a new document belongs, put day-to-day interaction
contracts in `partners/` and implementation, setup, deployment, or architecture
details in `developers/`.
